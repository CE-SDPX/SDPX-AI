# Presentation: Unit Testing

## รูปแบบ
- เวลา: **3 นาที** ต่อกลุ่ม (5 กลุ่มที่เหลือจากสัปดาห์ที่แล้ว)
- ผู้พูด: อย่างน้อย 2 คนในกลุ่ม
- แสดงผ่านหน้าจอ: terminal รัน tests + coverage report

---

## สิ่งที่ต้อง Present

1. **Show tests รันผ่าน** — เปิด terminal และรัน test suite ให้เห็นผล
2. **อธิบาย 1 test ที่น่าสนใจที่สุด** — บอกว่ามันวัดอะไร และทำไมถึงสำคัญ
3. **AI-generated tests** — บอกว่า AI generate test อะไรให้บ้าง เก็บไว้อะไร ทิ้งอะไร และทำไม

---

## Rubric การให้คะแนน (5 คะแนน)

| เกณฑ์ | 1 | 2 | 3 | 4 | 5 |
|---|---|---|---|---|---|
| **Tests รันผ่าน** | มี failure | ผ่านบางส่วน | ผ่านทั้งหมดแต่น้อย | ผ่านทั้งหมด ≥ 5 tests | ผ่านทั้งหมด ≥ 10 tests ครอบคลุม |
| **Test Quality** | แค่ test constructor | มี logic แต่ trivial | วัด business rule ได้ | วัด business rule + error cases | ครบ happy path + edge + error |
| **AI Review** | ไม่ได้ review AI output | review แต่เก็บทั้งหมด | ทิ้งบางส่วนแต่ไม่มีเหตุผล | ทิ้งพร้อมเหตุผล | ทิ้ง + ปรับปรุง + อธิบายได้ชัด |
| **Coverage** | ไม่มี coverage report | < 50% | 50-70% | 70-90% | > 90% และ identify gaps |
| **การนำเสนอ** | present ไม่ครบ | ครบแต่ไม่ชัด | ชัดเจนแต่คนเดียวพูด | ชัดเจน 2 คนขึ้นไป | ชัดเจน ตอบคำถามได้ทุกคน |

**คะแนนเต็ม: 5 คะแนน**

---

## คำถามที่อาจารย์อาจถาม
- "Test นี้จะ fail ได้อย่างไร — ช่วย break มันให้ดูหน่อย"
- "AI generate test อะไรให้บ้างที่กลุ่มตัดสินใจทิ้ง"
- "Coverage 80% ที่เหลืออีก 20% คือ code ส่วนไหน ทำไมยังไม่ test"
- "ถ้าต้อง refactor function นี้พรุ่งนี้ tests ที่มีจะช่วยได้ไหม"

# Present: First Deployment

## เวลา: 1 ชั่วโมง (6 กลุ่ม × 10 นาที)
## รูปแบบ
- อาจารย์ random 1 คนจากกลุ่มมา present — ไม่บอกล่วงหน้า
- ทุกคนในกลุ่มต้องพร้อมเสมอ
- ไม่ต้องมี slide — demo หน้าจอ

---

## สิ่งที่ต้อง Present (10 นาที/กลุ่ม)

### Demo (5 นาที)
1. เปิด live URL และแสดงให้เห็นว่า app ทำงานได้
2. เปิด GitHub repo แสดง structure และ README
3. แสดง branch `develop` และ commit history

### อธิบาย (3 นาที)
- Domain ที่เลือกคืออะไร แก้ปัญหาอะไรให้ใคร
- Stack ที่เลือกและเหตุผล
- ขั้นตอนที่ยากที่สุดใน lab วันนี้คืออะไร

### คำถาม (2 นาที)
อาจารย์อาจถาม:
- "ทำไมถึงเลือก domain นี้"
- "ถ้าเพื่อนในกลุ่มคนอื่นจะ clone repo นี้ไปรันต้องทำอะไรบ้าง"
- "AI ช่วยอะไรใน lab วันนี้ และคุณต้องแก้อะไรที่ AI สร้างมาบ้าง"

---

## Rubric (5 คะแนน)

| เกณฑ์ | 1 | 2 | 3 | 4 | 5 |
|---|---|---|---|---|---|
| **Live URL ทำงานได้** | URL ไม่ทำงาน | ทำงานแต่มี error | เปิดได้มีเนื้อหา | เปิดได้ สวยงาม | เปิดได้ + อธิบาย deploy process ได้ |
| **Repo Structure** | ไม่มี README | มี README แต่ว่าง | มี README บางส่วน | README ครบ | README + branch strategy + .gitignore ดี |
| **Domain Clarity** | ไม่ชัดว่าทำอะไร | พอเข้าใจแต่คลุมเครือ | เข้าใจได้ | ชัดเจน มี user | ชัดเจน มี user + pain point + value |
| **AI Usage** | ไม่ได้ใช้ AI | ใช้ AI แต่ตอบไม่ได้ว่า output ทำอะไร | อธิบายได้บางส่วน | อธิบาย AI output ได้ | อธิบาย + บอกได้ว่าแก้อะไรจาก AI output |
| **Presentation** | present ไม่ครบ | ครบแต่ไม่ชัด | ชัดเจน | ชัดเจน + ตอบคำถามได้ | ชัดเจน + ตอบทุกคำถาม + ดึงดูด |

**คะแนนเต็ม: 5 คะแนน**
